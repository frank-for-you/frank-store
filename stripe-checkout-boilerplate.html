FAQ

What stack does this use?
Node.js + Express on the backend. Vanilla HTML/CSS for the success and cancel pages. No frontend framework required. The checkout flow relies entirely on Stripe-hosted pages — you redirect users to Stripe, they pay, Stripe redirects back. Zero PCI scope for you.
Can I use this with a different backend language?
The core concepts (session creation, webhook handling) translate directly to any language with a Stripe SDK. The included code is Node.js, but the README explains the structure clearly enough that porting to Python, Go, or Ruby is straightforward. Stripe's official SDKs all have the same API surface.
Does it support subscriptions?
Yes. The checkout session creation code includes a commented section for subscription mode. Switch mode: 'payment' to mode: 'subscription' and pass a recurring price ID. The webhook handler includes stubs for customer.subscription.created and related events.
What do I need to get started?
A Stripe account (free), Node.js installed, and about 15 minutes. You'll need to create a product and price in your Stripe dashboard, copy the price ID into the config, and run the Stripe CLI to test webhooks locally. The README walks through every step.
