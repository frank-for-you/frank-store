What's planned

  • 🔄

    Agent Loop Scaffold

    A clean implementation of the observe-plan-act-evaluate loop. Handles context windowing, step limits, and graceful termination. The core structure you'd otherwise spend hours designing yourself.

  • 🔧

    Tool System

    Type-safe tool definitions with JSON Schema validation. Pre-built tools for file I/O, web fetching, code execution, and shell commands. Clean interface for adding your own tools in minutes.

  • 🧠

    Memory Layer

    Dual-mode memory: fast file-based storage for structured data, vector search for semantic recall. Handles context limits gracefully — important things persist, noise doesn't.

  • ⏱️

    Rate Limiting & Retries

    Exponential backoff with jitter, request queuing, and per-model rate limit tracking. Your agent won't die on a 429 or a transient API error.

  • 📋

    Structured Logging

    Every agent action, tool call, and decision logged in structured JSON. Easy to pipe to any log aggregator or just read in your terminal. Full audit trail out of the box.

  • 🐳

    Deployment Scaffold

    Dockerfile, docker-compose, and environment config that actually makes sense. Deploy to any VPS, Railway, Fly.io, or container service in under 10 minutes.

FAQ

Which AI provider does this support?
Primary support for the Anthropic Claude API (claude-sonnet-4-6 and claude-opus-4-6). The tool interface is designed to be provider-agnostic — adapters for OpenAI and Gemini are planned for a future update.
When will it ship?
Targeting Q2 2026. Follow the blog for updates — I'll post when it's in final testing and again when it launches.
What's the difference between this and just calling the API directly?
Calling the API directly gets you a single response. This kit handles the things that take hours to build properly: the agentic loop, context management, tool dispatch, memory persistence, error recovery, and deployment. It's the difference between a proof of concept and something you can actually run in production.
